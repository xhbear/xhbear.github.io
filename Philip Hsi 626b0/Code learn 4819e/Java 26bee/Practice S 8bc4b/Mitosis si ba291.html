Mitosis simulation

14 April, 2022

Length: 1 minute

Description:

Not an actual biology experiment!

Imagine a unique organism Beta that reproduces via mitosis.

bB = Big Beta

sB = Small Beta

Everyday, a Big Beta(bB) will split out 1 Small Beta(sB).

A Small Beta(sB) will also split out 1 Small Beta(sB).

Example:

On day 1, there is 1 bB.

On day 2, there will be 1bB + 1sB.

On day3, there will be 1bB + 3sB.

... and so on.

The program will input the current number of bB and sB. Your task is to return how many days has passed to reach the inputted population.

bB range: 1 ≤ PB ≤ 10

sB range: 0 ≤ NB ≤ 50,000

The tester will not input values that violates their reproductive behavior.

import java.util.Scanner;

public class Main {

    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        // 第一個小Beta一定來自大Beta
        // 大Beta的數量恆定
        // 第一天小Beta數量為零
        // 大Beta數量 = 小Beta第2天的數量
        // 從第三天開始，小Beta數量 = 自身*2+大Beta數量

        int bigBeta = sc.nextInt();
        int smallBeta = sc.nextInt();
        int days = 0;

        int iniOfBigB = bigBeta;
        int iniOfSmallB = 0;

        while (iniOfSmallB != smallBeta) {
            iniOfSmallB = iniOfSmallB * 2 + bigBeta;
            days++;
        }
        System.out.println(days);
    }
}

My thoughts:

  1. The first batch of Small Betas must come from Big Betas.
  1. And that through out the observation period, the number of Big Betas will remain constant.
  1. Keeping the above logical relationship in mind, it can be deduced that there must be no Small Betas within the population on the first day.
  1. And that the number of Small Betas on day 2 must be equal to the number of Big Betas.
